About This Course

This course is designed to teach students how to work with Git and GitLab for a team project. Through hands-on exercises, you'll learn the essential skills needed for collaborative software development in a professional environment.

Prerequisites

This course requires a basic understanding of:

  • Git - Basic commands like add, commit, push, and pull
  • Python - Basic programming concepts and syntax

Team Roles and Collaboration

Each member of the team should pick a role (A, B, C, D, E, or F) and perform the instructions described for that role. This course is designed for teams of 6, but can be adapted for smaller teams.

Important: Some tasks require you to wait until a teammate has finished theirs. Pay attention to dependencies between tasks!

For teams with fewer than 6 members: All roles are necessary for the course. If you have fewer than 6 members, some team members will need to play two roles to complete all required tasks.

Course Modules

Module 0: Setting Up the Working Environment

Learn how to set up your development environment with all the necessary tools. This includes installing VSCode, Git, setting up SSH keys, creating and cloning repositories, setting up Python virtual environments, and making your first commits.

Module 1: Working in Parallel Without Conflicts

Discover how teams can work simultaneously on different parts of a project without stepping on each other's toes. You'll learn about project structure, task splitting, creating merge requests, and setting up branch protection.

Module 2: How to Deal with Conflicts

Learn the essential skill of resolving merge conflicts when team members work on overlapping code. This module covers identifying conflicts, resolving them properly, and deploying the merged code to the main branch.

Module 3: Best Practices for Having a Clean Project

Master the best practices that professional teams use to maintain clean, well-structured projects. This includes setting up GitLab CI/CD pipelines and organizing your Python code with proper package structure.
