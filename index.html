Course Overview

Throughout this course, your team of 6 students will collaboratively develop a Python hangman game using professional Git and GitLab workflows. You'll start by setting up your development environment and learning the basics of repository management.

As you progress, each team member will be assigned specific components of the hangman game to implement - from core game logic and user interface to testing frameworks and documentation. You'll learn to work in parallel using feature branches, handle merge conflicts when they arise, and conduct code reviews through GitLab's merge request system.

By the end of the course, you'll have built a complete, tested hangman game while mastering the collaborative development practices used in professional software teams. The project includes unit testing, continuous integration pipelines, and follows Python best practices for package structure and code quality.

Duration: ~6 hours across 6 modules Team Size: 6 students per group Format: Role-based collaborative learning
